* 프로젝트의 목표

+ 차기 게임 제작 (차세대가 아님) 을 위한 기반 기술 확립 및 연구
+ 게임성 향상을 위한 기반 기술 연구
+ 아울러 신규 게임 제작을 위한 새로운 게임성 발견

- 프로젝트의 완성은 중요한가? - 완성은 중요하다. 하지만 프로젝트 자체의 완성도는 상대적으로 중요도가 낮다
- 프레임웍은 얼마나? - 테스트 프로젝트에 가깝기 때문에, 알고리즘 구현과 모듈화가 그 목표이다.   
  물론 이대로 프레임웍까지 갖춰진다면 좋겠지만, 한 번에 만족할만큼의 프레임웍 구축은 어렵다.

* 프로젝트의 기획적 목표

+ 보다 재미있는 전투 구현(AI)
+ 보다 응용가능한 액션전투 시스템 구현
+ 대형 몬스터의 액션적 MMO 파티전투 구현

* 프로젝트의 그래픽적 목표

+ 독창적 분위기의 실험 및 기술적 확립
+ 효율성 증대를 위한 기술적 연구

* 프로젝트의 기술적 방향 (milestone)

+ '구현되는' 기본적 이동과 공격. 배경, 카메라 뷰 구현
+ 액션감 / 타격감 상승.
+ 멀티 타겟 캐릭터
+ 캐릭터의 가드 개념 추가
+ 캐릭터의 지역 공격 추가
+ 몬스터의 인공지능 : 어그로, 자기방어
+ 오브젝트의 파괴/ 다이나믹 라이트

* 1차 마일스톤

+ 캐릭터의 이동, 공격, 대기, 죽기등의 기본 애니메이션 적용
+ 카메라 뷰 구현
+ 배경 기본 구현
+ 그래픽 요소는 최소화 시킴
+ 일반적 게임과 다를 바 없는 수준
+ 뷰와 게임의 스타일 확립
+ 상하체 분리 애니메이션, 무빙어텍 구현여부?

* 2차 마일스톤

+ 액션감/ 타격감에 대한 고찰
+ 타격감이란 무엇인가?

+ 주인공 캐릭터 지역 공격 추가
+ 멀티 타겟 캐릭터 제작
